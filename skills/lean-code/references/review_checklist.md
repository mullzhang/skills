# コードレビューチェックリスト

コード生成後、以下の項目を確認する。1つでも該当すれば修正が必要。

## 過剰防御チェック

- [ ] 到達不能なフォールバックパスがないか（存在しない旧形式への対応など）
- [ ] `try-catch` の中で、発生しないはずの例外を握りつぶしていないか
- [ ] 「念のため」のデフォルト値が、バグを隠す原因になっていないか
- [ ] 条件分岐の中に、現在のコードベースでは絶対に通らないパスがないか

## DRY チェック

- [ ] 同一または酷似したロジックが2箇所以上に存在しないか
- [ ] コピペして微修正したような関数がないか
- [ ] 共通化できる処理がインラインで展開されていないか

## YAGNI チェック

- [ ] 現時点で呼び出し元が存在しない関数・メソッドがないか
- [ ] まだ存在しない要件に対する抽象化レイヤーがないか
- [ ] 使われていない引数・オプション・設定がないか
